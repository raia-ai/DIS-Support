_Create a Thermo King API Role - Altura _

Introduction 

 User Roles in Altura are created as templates and control whether users can Create, Read, Update, and/or Delete information. This article details how to create a custom Thermo King API Role in Altura that allows users to view TK Bundle 1 information such as Equipment Verification, Warranty, Campaign Data, and Service History. 

   

 Create the Thermo King API Role 

 Important! When assigning a customized Role to a user, all permissions configured for that Role are automatically applied to the user. 

 Select Roles & Permissions (Admin Settings | Staff | Roles & Permissions) . 

 Click the + Role button in the top-left corner of the screen. 

 Enter Thermo King API for the Role title in the Title field. If desired, provide a description. 

 Click the Add Role button. 

 

 Click the Thermo King API accordion to reveal the available permissions. 

 Slide the toggle buttons to the right to enable permissions for the new Role. Enabled permissions will display in blue as shown above. 

 See the Guide to Users, Roles, and Permissions article for additional information on Roles in Altura.
